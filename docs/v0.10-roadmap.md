## 待做（按 Koko 优先级）

### 🔴 **P0 — 记忆系统重做（待视觉调参稳定后启动）**
> "记忆才是最贴近灵魂的"

**核心问题**：
- oc 压缩上下文后，对话怎么接着走
- 记忆拉取格式有问题（Koko 原话）

**当前状况**：
- korina `plugins/memory/`（57 行，仅占位）
- 历史：`memory.checkpoint` 顺序、summarize 失败也继续 recall（v0.3 路线图遗留）
- 外部脚本：`~/search_oc_memory.py`（Agnes API + 向量检索）
- 痛点：compressor 输出格式 / recall 注入格式 / 跨 session 对接 都没标准化

**调研清单**（启动前先看）：
- [ ] oc 压缩触发条件（自动？手动？token 阈值？）
- [ ] 压缩后 prompt 长什么样（summary? structured?）
- [ ] korina memory plugin 的 recall 输出格式
- [ ] AI 拿到 compressed context 后的真实表现
- [ ] search_oc_memory.py 当前 recall 格式 vs AI 期望格式

**设计方向（待定，启动时确认）**：
- 让记忆成为**显式一等公民**——不只 recall，还要 structure
- 每次 heartbeat 注入的 memory 块格式稳定（schema 化）
- 跨 session 的"主线程"识别（哪些 session 是连续的）
- 记忆=对话能力的支柱，框架只是载体
