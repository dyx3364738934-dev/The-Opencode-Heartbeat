## v0.2 → v0.2.1 已有成果（dogfooded + verified）

### 核心修复
- `.git/` 长轮询 → file-watcher 硬过滤
- 自身 src/ 长轮询 → 硬过滤
- `furina-main.log` 丢失 → watchdog + plugin 都加 stdout/stderr 重定向
- watchdog 密码过期逻辑 → 放宽到 verify_server 验证
- session 锁定重启丢失 → `logs/session.lock` 持久化 + 自动恢复
- `set-watch` 重复添加 → 路径去重
- `memory.checkpoint` 顺序 → summarize 失败也继续 recall
- `injector._poll` 等不到 oc completed → 60s 软超时 + 双稳定判断
- `injector.inject` 10s 超时不够 → 30s race timeout + fire-and-forget

### 新增能力
- `sensors/timer-sensor.mjs`（周期感知器）
- `memory.recall()` 控制通道命令
- 持久化 session 自动恢复
- `presets.json` 加 timer + watchPath 配置

## v0.3 目标

### 1. 真正的 AI 自主任务生成（dogfooding 闭环）
- 当前：`findWorkPrompts` / `selfTalkPrompts` 是固定文本
- v0.3：让 oc 自己在 idle 时生成下一步任务（基于 `search_oc_memory.py` 检索的工作记忆）
- 实现：`mode-manager._findWork()` 改成 inject "请基于近期记忆决定下一步任务"

### 2. 跨 session 记忆共享
- 当前：每个 session 独立，furina 通过 `recentRecall` 跨 session 续接
- v0.3：在 dispatchHandler 触发时，主动检索 `search_oc_memory.py --query "{recent activity}"`，把高相关片段注入到 message
- 避免"对话上下文丢失"

### 3. 感知器生态扩展
- `git-sensor.mjs`：监听 `~/Desktop/<project>/.git/HEAD` 等 ref 变化（已硬过滤 .git/，但主动监听 git 状态可获取 commit 信息）
- `process-sensor.mjs`：监听特定进程启停（oc.exe, watchdog.exe 等）
- `network-sensor.mjs`：监听端口变化（furina 启动时找到的 oc 端口如果变了能自动重连）

### 4. watchdog + plugin 协作重构
- 当前：watchdog 兜底启动 furina，但 plugin 也启动 furina，两者可能冲突
- v0.3：watchdog 信任 plugin，plugin 失败时 watchdog 才介入；通过共享锁文件（`logs/furina.lock`）避免重复启动

### 5. 自我监控与报告
- 当前：furina 自己监控自己（heartbeat.json），但没有主动汇报
- v0.3：furina 在状态异常时（queue 积压、tokens 耗尽、oc 持续 backlog）自动 inject 警告给当前 session
- 让 Koko 回来时能立即看到 furina 的"健康状况"

### 6. 配置文件 schema 化
- 当前：`presets.json` 是纯 JSON，没 schema 验证
- v0.3：用 JSON Schema 或 Zod 验证，写错时给清晰错误
- 加 `config/schema.json` 描述

### 7. 测试覆盖
- 当前：`tests/test-e2e.mjs` 是端到端测试
- v0.3：
  - unit test for `_poll`（mock fetch）
  - unit test for `event-queue.mjs`（令牌桶、去抖）
  - unit test for `file-watcher.mjs`（chokidar mock）
  - integration test for `checkpoint` → `recall` 链路

## 不做（明确边界）

- ❌ 改 oc 桌面版的 prompt_async 行为（furina 是外部辅助，不改 oc）
- ❌ 让 furina 自己写代码不通知 Koko（自我迭代要先在 lockfile 里记录，给 Koko 完整审计轨迹）
- ❌ 跨设备同步（furina 是单机的，多机同步需要 oc 重写）

## 时间估算

| 任务 | 难度 | 估时 |
|------|------|------|
| 1. 自主任务生成 | 中 | 2-3h |
| 2. 跨 session 记忆 | 低 | 1h |
| 3. 感知器扩展（git/process/network） | 中 | 每个 1h |
| 4. watchdog/plugin 重构 | 中 | 2h |
| 5. 自我监控报告 | 低 | 30min |
| 6. schema 化 | 低 | 1h |
| 7. 测试覆盖 | 中 | 4-6h |
